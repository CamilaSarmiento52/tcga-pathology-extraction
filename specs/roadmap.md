### Baseline Findings & Gaps (2026-05-31)

All three business success criteria **FAIL** for both models at this baseline. This is expected for a first-pass evaluation; gaps below drive Phase 5+ improvements.

#### openai:o4-mini
| Criterion | Result | Target | Status |
|-----------|--------|--------|--------|
| Primary: per-field F1 thresholds | Worst fields: `primary_site` F1=0.616, `specimen_type` F1=0.432, `tumor_grade` F1=0.503 | Low≥0.89, Med≥0.83, High≥0.87 | FAIL |
| Secondary: hallucination rate | 15.00% | ≤5% | FAIL |
| Tertiary: pipeline reliability | 83.89% (151/180 valid) | ≥95% | FAIL |

**Root causes identified:**
- `primary_site` and `specimen_type` low F1: vocabulary mismatch — model outputs free-text variants not covered by the clinical vocab list, triggering `vocab_flagged` status and mismatches against ground truth
- `tumor_grade` low F1: grading schemes vary by cancer type (Nottingham for BRCA, IASLC for lung); the prompt does not instruct the model which scheme to apply per cancer type
- High hallucination rate (15%): model populates fields (especially `tnm_stage.M`) with inferred values absent from the report text; `tnm_stage.M` absent in ~89% of BRCA reports is clinically normal but currently counted as hallucination

#### ollama:llama3.2
| Criterion | Result | Target | Status |
|-----------|--------|--------|--------|
| Primary: per-field F1 thresholds | All fields near 0 (best: `histological_diagnosis` F1=0.092) | Low≥0.89, Med≥0.83, High≥0.87 | FAIL |
| Secondary: hallucination rate | 4.29% | ≤5% | PASS |
| Tertiary: pipeline reliability | 31.11% (56/180 valid) | ≥95% | FAIL |

**Root causes identified:**
- 69% structural failure rate (`json_failed` + `schema_failed`): llama3.2 (3B) cannot reliably produce schema-conformant JSON for this extraction task; the model is too small for structured extraction at this complexity level
- Near-zero F1 on all fields: only 56 records parse successfully, and even those show poor field accuracy
- llama3.2 is **not recommended** for production use; a larger GDPR-compliant model (e.g. `ollama:llama3.1:70b` or a hosted EU-region model) should be evaluated in Phase 5

#### Improvement tasks for next phases
- [ ] Expand clinical vocab lists for `primary_site` and `specimen_type` to cover TCGA free-text variants
- [ ] Add cancer-type-aware grading instructions to prompt (Nottingham for BRCA, IASLC for LUAD/LUSC)
- [ ] Revise hallucination detection: exclude `tnm_stage.M` null from hallucination count (clinically normal absence)

### Phase 7 Results (2026-06-20)

**Few-shot budget finding — 12-shot hurts:** Increasing from 5-shot to 12-shot degraded all models. Mistral Nemo combined_score dropped from **0.77 → 0.66** and hallucination rate jumped from **15% → 41%**. o4-mini hallucination rate also increased at 12-shot. Hypothesis: 12 examples push the prompt into the model's noise zone — longer context increases copying/confabulation rather than calibration. **Recommendation: fix few-shot count at 5 for all future runs.**

**Baseline ranking (5-shot):**

| Model | combined_score | hallucination_rate (5-shot) | hallucination_rate (0-shot) | Context window |
|---|---|---|---|---|
| openai:o4-mini | highest | lower than Mistral Nemo | — | 128k |
| ollama:mistral-nemo | 0.77 | 15% | 30% | 128k |
| ollama:openbiollm | 0.20 | — | — | 8k ⚠️ ❌ dropped |

**OpenBioLLM — dropped from roster (Phase 8, 2026-06-23):** Six experiments confirmed the model cannot reliably extract clinical content regardless of shot count, prompt style, or endpoint. Root causes: (1) 8k context ceiling makes most prompts hit the limit at ≥2-shot, (2) instruction-following failure returns null skeletons even on short 2,300-token prompts. Grammar-constrained JSON (native `/api/chat`) eliminates free text but cannot fix extraction failure. See `specs/features/phase-8-improvements/plan.md` for full decision record.

**Hallucination pattern:** All models show 0-shot hallucination rates significantly above 5-shot, except the direction reversed at 12-shot (rate went back up). This is consistent with few-shot examples providing calibration up to a point, after which longer prompts introduce noise. Mistral Nemo hallucination rate is systematically higher than o4-mini across all shot counts.

**Hardest field — histological_subtype:** Consistently the weakest field across all models. SapBERT similarity scores are low; suspected cause is that subtype vocabulary is highly specialised, uses long multi-word expressions, and the model outputs surface forms that SapBERT cannot bridge (e.g., acronyms vs spelled-out forms). Normalisation may be insufficient.

**tumor_grade not using semantic similarity:** Currently scored by exact string match rather than SapBERT. Given the variety of valid grade notations (NHG, G1-G4, "poorly differentiated", range forms), this is a meaningful gap — a correct-but-differently-worded grade gets zero score. Flagged for next phase.

**primary_site hard-coded rules:** The current scorer applies explicit string rules before SapBERT. These are brittle and opaque. Preference is to rely on pure cosine similarity from SapBERT embeddings and remove the hand-coded overrides.

#### Phase 9 (after Phase 8 settles)

**Priority order for Phase 9:**

1. **AJCC-based clinical validation** *(before langextract)* — use the AJCC 7th and 8th edition staging manuals already in `AJCC/` to build a per-cancer-type validator that flags:
   - TNM combinations that are impossible or inconsistent for the given cancer type (e.g. M-stage codes that don't exist for BRCA, T/N combinations outside allowed ranges)
   - Grade × histological type mismatches (e.g. Grade 4 for squamous cell carcinoma, which uses a 3-tier system)
   - Histological diagnosis × primary site mismatches (e.g. "infiltrating lobular carcinoma" at a lung site)

2. **langextract (source grounding / hallucination v2)** — `google/langextract` character-level provenance: maps each extracted value back to a character span in the source text, replacing vocab-based `hallucination_flags` with genuine "was this value found in the report?" verification. 3. **Pipeline simplification** — once Phase 8 model and embedding changes are stable, audit what is load-bearing and reduce accidental complexity: (1) single vocabulary source of truth shared by `validator.py` and `semantic.py`, (2) consolidate evaluation notebooks, (3) fold `model_comparison.py` into `metrics.py`, (4) replace the hand-maintained GDPR dict with a model config file.
